# Design Token Reference

Design tokens are CSS custom properties in `.ui-specs/tokens.css`, shared across all page specs.

## Token Categories

### Colors (10 tokens)

| Token | Purpose | Derivation |
|-------|---------|------------|
| `--color-primary` | Brand primary, CTAs, active states | From spec.md §11 design direction |
| `--color-secondary` | Supporting actions, secondary buttons | Complement of primary |
| `--color-accent` | Highlights, badges, notifications | Contrast to primary |
| `--color-bg` | Page background | Light: `#fafafa`, Dark: `#0f0f0f` |
| `--color-surface` | Cards, panels, modals | Light: `#ffffff`, Dark: `#1a1a1a` |
| `--color-text` | Body text | Light: `#1a1a1a`, Dark: `#f0f0f0` |
| `--color-text-muted` | Captions, placeholders | 50% opacity of `--color-text` |
| `--color-border` | Dividers, input borders | Light: `#e2e2e2`, Dark: `#2a2a2a` |
| `--color-error` | Errors, destructive actions | `#dc2626` |
| `--color-success` | Confirmations, positive states | `#16a34a` |

### Typography (6 tokens)

| Token | Purpose | Default |
|-------|---------|---------|
| `--font-display` | Headings, hero text | `'Inter', sans-serif` |
| `--font-body` | Paragraphs, labels, UI text | `'Inter', sans-serif` |
| `--font-mono` | Code, technical values | `'JetBrains Mono', monospace` |
| `--font-size-base` | Body text size | `16px` |
| `--font-size-sm` | Captions, helper text | `14px` |
| `--font-size-lg` | Subheadings | `20px` |

### Spacing (6 tokens)

| Token | Formula | Default |
|-------|---------|---------|
| `--space-unit` | Base unit | `8px` |
| `--space-xs` | `unit * 0.5` | `4px` |
| `--space-sm` | `unit * 1` | `8px` |
| `--space-md` | `unit * 2` | `16px` |
| `--space-lg` | `unit * 3` | `24px` |
| `--space-xl` | `unit * 4` | `32px` |

### Layout (4 tokens)

| Token | Purpose | Default |
|-------|---------|---------|
| `--radius-sm` | Inputs, small elements | `4px` |
| `--radius-md` | Cards, panels | `8px` |
| `--radius-lg` | Modals, hero sections | `16px` |
| `--shadow-md` | Elevation for cards/dropdowns | `0 4px 6px rgba(0,0,0,0.1)` |

## Token File Format

```css
/* tokens.css — Generated by /ui-specs */
@import url('https://fonts.googleapis.com/css2?family={display}:wght@400;600;700&family={body}:wght@400;500;600&display=swap');

:root {
  /* Colors */
  --color-primary: {value};
  --color-secondary: {value};
  /* ... all 10 color tokens */

  /* Typography */
  --font-display: '{name}', sans-serif;
  /* ... all 6 typography tokens */

  /* Spacing */
  --space-unit: {value};
  /* ... all 6 spacing tokens */

  /* Layout */
  --radius-sm: {value};
  /* ... all 4 layout tokens */
}
```

## Deriving Tokens from spec.md §11

| Design Direction Keyword | Token Mapping |
|--------------------------|---------------|
| "minimal", "clean" | Neutral palette, generous spacing (`--space-unit: 8px`), large radius |
| "bold", "vibrant" | Saturated primary, tight spacing, small radius |
| "corporate", "professional" | Blue/gray palette, system fonts, medium spacing |
| "playful", "fun" | Warm colors, rounded radius (`--radius-md: 12px`), display fonts |
| "dark mode" | Invert bg/surface/text defaults |

## BAD vs GOOD Token Extraction

| BAD | GOOD | Why |
|-----|------|-----|
| Skip tokens, hardcode `#3498db` everywhere | Define `--color-primary: #3498db` and reference | Gallery can't edit hardcoded values |
| `--color-1`, `--color-2` | `--color-primary`, `--color-accent` | Semantic names communicate purpose |
| 50 granular tokens | 26 tokens (10+6+6+4) | Manageable set, covers all needs |
| Derive tokens without asking user | Propose defaults, confirm via AskUserQuestion | User owns final design decisions |
